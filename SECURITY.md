# 安全策略

## 支持范围

安全修复面向当前 main 分支和最新发布版本。历史版本可能只在最新版本中获得修复。

## 私密报告

请使用 GitHub 仓库 Security 页面中的 private vulnerability reporting 提交漏洞。报告应包含受影响版本或 commit、影响、最小复现、可能的缓解方式，以及可安全共享的日志。

不要通过公开 issue、discussion 或 pull request 披露漏洞细节、有效利用代码、凭据或个人数据。

如果 private vulnerability reporting 入口不可用，可以公开提出“需要私密安全联系渠道”的请求，但不要附带任何漏洞细节。维护者会提供私密后续方式。

普通渲染错误、构建失败和功能建议请使用 bug 表单。CUDA、OptiX、NVIDIA 驱动或其他上游组件的问题也应按相应供应商的安全流程报告。
